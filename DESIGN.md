# EasyFocus 功能梳理与 QuickFocus 设计

## EasyFocus 的实际工作流程

1. 插件加载后读取角色配置。
2. 根据标记、频道和喊话模板生成焦点宏。
3. 可选生成第二个清除焦点宏。
4. 对安全单位框安装“修饰键 + 左键”的宏点击属性。
5. 用隐藏 `SecureActionButtonTemplate` 接住无单位区域的相同按键。
6. 战斗中不修改安全绑定，设置变更延迟到脱战后应用。

因此它不依赖 Clique，也不使用轮询。

## 保留的用户功能

| 功能 | QuickFocus |
| --- | --- |
| 鼠标悬停敌对单位设焦点 | 保留 |
| 自动团队标记 | 保留 |
| 自定义修饰键 | 保留 |
| 多频道及自定义频道喊话 | 保留 |
| 喊话占位符 | 保留 |
| 空白处清除焦点和标记 | 保留，可选 |
| 战斗中安全延迟应用 | 保留 |
| 图形设置页 | 保留，延迟创建 |
| 状态命令 | 保留并简化为 `/qf status` |

## 重写后的简化点

- 删除无实际扫描行为的“重新扫描单位框”概念，改为事件驱动安装安全点击属性。
- 删除常驻宏预览控件和重复的宏重建流程。
- 配置、宏生成、安全点击和界面职责分离。
- 所有选择项使用紧凑数组，减少重复表字段。
- 设置页只注册一个空面板，首次打开时才创建控件。
- 多次连续修改只排队一次刷新。
- 只需要 `QF_Focus`，空白处清除由同一个宏内的条件语句处理。
- 不创建 OnUpdate，不持续遍历单位框，不监听战斗日志。

## 为什么仍然使用真实宏

焦点、团队标记等属于受保护动作。新版客户端限制插件动态创建并执行
`macrotext`，因此隐藏安全按钮不能完全替代真实宏。QuickFocus 继续使用真实宏，
再由安全点击属性触发，这是兼容战斗状态且行为稳定的实现。

## 运行时资源模型

- 事件：`ADDON_LOADED`、`PLAYER_LOGIN`、`PLAYER_ENTERING_WORLD`、`UPDATE_MACROS`、`NAME_PLATE_UNIT_ADDED`、`GROUP_ROSTER_UPDATE`、`PLAYER_REGEN_ENABLED`
- 安全按钮：1 个
- 计时器：仅设置变更时创建一次零延迟合并刷新
- 常驻轮询：0
- 单位框钩子：`CreateFrame` secure hook，脱战后安装属性
- 默认宏槽：1
